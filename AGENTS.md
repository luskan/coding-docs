## Load-bearing facts the docs must keep accurate (verified in the app)

1. Dagger 2.60 dropped transitive `error_prone_annotations`; every module with a real `@Inject` needs
   `compileOnly("com.google.errorprone:error_prone_annotations:2.36.0")` or the first injection fails
   with `package com.google.errorprone.annotations does not exist`.
2. `gradle.properties` needs `android.disallowKotlinSourceSets=false` (KSP + AGP 9 built-in Kotlin).
3. `hiltViewModel()` lives in `androidx.hilt:hilt-lifecycle-viewmodel-compose` /
   `androidx.hilt.lifecycle.viewmodel.compose` (since 1.3); the old `hilt-navigation-compose` is deprecated.
4. Assisted ViewModel needs explicit type args: `hiltViewModel<VM, VMF>(creationCallback = { it.create(...) })`.
5. A `@Provides`/`@Binds` method name can't be a Java keyword (`default` fails KSP -- Dagger emits Java).
6. Scope validation (`IncompatiblyScopedBindings`) only fires for **reachable** bindings.
7. `@HiltWorker` needs two KSP processors: Dagger's `hilt-android-compiler` AND androidx `hilt-compiler`.
8. Multi-module: one `@HiltAndroidApp` (in `:app`); an `internal` impl in a library module is
   unreachable from other modules but its `@Binds` still merges into the single app-wide graph.
